# 超级大脑交易系统 — 信号评分与开仓逻辑参考手册

> 版本：2026-03-19
> 适用服务：U本位合约（`smart_trader_service.py`）
> 核心类：`SmartDecisionBrain.analyze()`

---

## 一、整体评分架构

```
K线数据 (kline_data)
    ↓
[V1 信号评分] SmartDecisionBrain.analyze()
    ├── 14个技术信号组件 → long_score / short_score
    ├── 动态阈值（基于Big4状态）
    └── 信号合格性校验（方向一致性、最少组件数等）
    ↓
[V2 协同过滤] SignalScoreV2Service.check_score_filter()
    ├── coin_kline_scores 表（每5分钟更新）
    └── 反向冲突检测（opposite strength >= 20 → block）
    ↓
[Big4 趋势过滤] Big4TrendDetector.detect_market_trend()
    ├── BTC/ETH/BNB/SOL 四币加权判断
    └── 强度>=40 时 veto 逆势方向
    ↓
[羊群效应防御] herding filter
    └── Big4=NEUTRAL 时，80%+同向 → LONG+5惩罚 / SHORT全放行
         （注：强牛市时Big4=BULLISH，herding filter不激活，LONG羊群天然放行）
    ↓
开仓执行
```

## 二、V1 信号组件详解（14个活跃组件）

权重来源：`signal_scoring_weights` 表（每日00:00 UTC自动优化，范围[5,30]）

### 2.1 仓位类组件

| 组件名 | 触发条件 | LONG | SHORT | 备注 |
|--------|---------|------|-------|------|
| `position_low` | `position_pct < 30` 且 `net_power_1h > -2` | 20 | 0 | 低位做多（需排除强力空头量能） |
| `position_mid` | `30 ≤ position_pct ≤ 70` 且 `\|net_power_1h\| >= 2` | 5 | 5 | 中位需方向性量能支撑；需3+组件才合格 |
| `position_high` | `position_pct > 70` | 0 | 20 | 高位做空；Big4=BULLISH时自动抑制 |
| `breakdown_short` | `position_pct < 30` 且 `net_power_1h ≤ -2` 且 `net_power_15m ≤ -2` | 0 | **30** | 低位破位做空（双周期量能确认）；权重30与position_high等级 |

> `position_pct`：价格在100H最高/最低区间内的百分位
> `net_power`：24H内强力多头K线数 − 强力空头K线数（强力 = 收盘方向 + 成交量>均量×1.5）

## 三、动态开仓阈值

### 3.1 Big4过滤器 **开启**时（`big4_filter_enabled=true`）

| Big4状态 | LONG阈值 | SHORT阈值 | 说明 |
|---------|---------|---------|------|
| BULLISH（强度≥50） | 60 + 群体惩罚 | 60 | 顺势做多 |
| BEARISH（强度≥50） | **100**（实质封死） | 60 | 熊市禁止做多 |
| NEUTRAL | 65 | 60 | 中性适度保守 |

**群体惩罚**（Big4非STRONG_*时才生效）：
- 强度≥70 → +15分
- 强度≥55 → +8分
- STRONG_BULLISH/STRONG_BEARISH → +0（真实趋势，不惩罚）

### 3.2 Big4过滤器 **关闭**时（`big4_filter_enabled=false`，当前测试模式）

**软调整模式**（commit 594a06d，2026-03-19）：

| Big4状态 | LONG阈值 | SHORT阈值 | 说明 |
|---------|---------|---------|------|
| BEARISH（强度≥50） | **75** | 60 | 软限制：过滤低分逆势多单 |
| BULLISH（强度≥50） | 60 | **75** | 牛市中追空风险高，提高门槛 |
| NEUTRAL | 60 | 60 | 双向正常，完整采集信号数据 |

> ⚠️ Hard veto（执行层屏蔽）在过滤器关闭时**不生效**，软调整仅影响阈值

## 四、信号合格性校验（开仓前5道门）

V1评分完成后，信号还需通过以下校验才进入候选：

```
Gate 1: 最少组件数
  len(signal_components) >= 2，否则拒绝
  position_mid 若存在，需 len >= 3

Gate 2: 方向一致性
  LONG信号中不能包含空头组件（除非特例：position_low+momentum_down_3pct组合）
  SHORT信号中不能包含多头组件（除非特例：position_high+momentum_up_3pct组合）

Gate 3: 仓位-方向矛盾检查
  LONG 时不允许含 position_high
  SHORT 时不允许含 position_low

Gate 4: 最高分拒绝门
  score > 150 → 拒绝（历史数据：150+分胜率仅32.8%，平均亏-4.82U/单，追涨杀跌特征）

Gate 5: 信号黑名单
  查询 signal_blacklist 表，匹配 (signal_combination, position_side) → 拒绝
```

## 五、V2 协同过滤逻辑

**数据来源**：`coin_kline_scores` 表（每5分钟更新，总分范围约 -100 ~ +100）

```python
# 数据新鲜度检查
if last_updated > 15分钟前:
    自动通过（数据过期，不阻拦）

# 反向冲突检测
if V2_direction != V1_direction AND abs(V2_score) >= 20:
    如果 V1_score >= 阈值 + 25:
        记录警告日志，允许开仓（强信号穿透V2阻拦）
    else:
        拒绝（V2反向且强度足够，视为真实冲突）

# NEUTRAL/弱同向 → 通过
```

**V2评分计算规则**（`coin_kline_scores` 表）：
```
h1_score:
  1H阳线≥18根(60%) → +40 STRONG
  1H阳线≥16根(53%) → +30 MEDIUM
  1H阴线≥18根      → -40 / ≥16根 → -30

m15_score:
  15M阳线≥11根(69%) → +30 / ≥9根(56%) → +20
  15M阴线≥11根      → -30 / ≥9根      → -20

five_m_bonus（反向K线确认）:
  主趋势为多 + 5M出现阴线(回调) → +abs(5M_score)
  主趋势为空 + 5M出现阳线(反弹) → +abs(5M_score)

total_score = h1_score + m15_score + five_m_bonus
```

## 六、Big4 趋势判断逻辑

**权重**：BTC 50% | ETH 30% | BNB 10% | SOL 10%
**文件**：`app/services/big4_trend_detector.py`

### 6.1 单币分析算法

每个币种独立分析，使用：
- **1H K线 30根**（主趋势）
- **15M K线 16根**（确认）
- **5M K线 3根**（反向确认加分）

```
h1_score计算（30根1H K线）：
  阳线≥18根(60%) → +40  level=STRONG
  阳线≥16根(53%) → +30  level=MEDIUM
  阴线≥18根(60%) → -40  level=STRONG
  阴线≥16根(53%) → -30  level=MEDIUM
  其他           →  0   level=NEUTRAL

m15_score计算（16根15M K线）：
  阳线≥11根(69%) → +30  level=STRONG
  阳线≥9根 (56%) → +20  level=MEDIUM
  阴线≥11根(69%) → -30
  阴线≥9根 (56%) → -20
  其他           →  0

5M反向确认加分：
  h1+m15合计 > 0（多头趋势）且 5M阴线 → +abs(5M_score)  // 回调确认
  h1+m15合计 < 0（空头趋势）且 5M阳线 → +abs(5M_score)  // 反弹确认

单币最终信号：
  signal_score = h1_score + m15_score + 5m_bonus
  score >=  50 → BULLISH
  score <= -50 → BEARISH
  其他         → NEUTRAL
```

### 6.2 四币加权综合判断

```python
bullish_weight = sum(weight for 各BULLISH币种)  # BTC=0.5, ETH=0.3, BNB=0.1, SOL=0.1
bearish_weight = sum(weight for 各BEARISH币种)
avg_strength   = Σ(coin_signal_strength × coin_weight)

# 判断规则（优先级由高到低）：
if bullish_weight > 0.70 AND avg_strength > 70:
    overall = 'STRONG_BULLISH'
elif bearish_weight > 0.70 AND avg_strength > 70:
    overall = 'STRONG_BEARISH'
elif bullish_weight > 0.60:
    overall = 'BULLISH'
elif bearish_weight > 0.60:
    overall = 'BEARISH'
else:
    overall = 'NEUTRAL'
```

**权重组合举例**：
- BTC+ETH = 0.80 → BULLISH（>0.60 ✓）
- BTC单独 = 0.50 → NEUTRAL（不够0.60）
- ETH+BNB+SOL = 0.50 → NEUTRAL（不够0.60）
- 全部四币多 = 1.00 + 高强度 → STRONG_BULLISH

### 6.3 震荡市场检测

```python
# 检测近4小时 big4_trend_history 记录
bullish_count = 多头记录数
bearish_count = 空头记录数
ratio = min(bullish, bearish) / max(bullish, bearish)

is_choppy = (ratio > 0.40)  # 多空来回翻 = 震荡市
```

### 6.4 Big4 Veto 规则（执行层硬封，Big4过滤器开启时生效）

```python
# 强度>=40 时触发逆势封锁（注意：STRONG_*变体必须包含在判断中）
if big4_signal in ('BULLISH', 'STRONG_BULLISH') AND big4_strength >= 40 AND side == 'SHORT':
    REJECT "禁止逆势做空"

if big4_signal in ('BEARISH', 'STRONG_BEARISH') AND big4_strength >= 40 AND side == 'LONG':
    REJECT "禁止逆势做多"
```

> ⚠️ **常见陷阱**：`== 'BULLISH'` 不能匹配 `STRONG_BULLISH`！必须用 `in ('BULLISH', 'STRONG_BULLISH')`
> 修复记录：commit fa2c743（2026-03-16），同时修复 smart_trader_service.py 和 coin_futures_trader_service.py 共6处

## 七、羊群效应防御（Herding Filter）

**触发条件**：Big4=NEUTRAL 且 候选信号 ≥ 5个 且 同向占比 ≥ 80%

```python
if dominant_side == 'LONG':
    # 恐慌性追涨 → 提高LONG门槛
    herding_threshold = base_threshold + 5   # 60 → 65
    过滤掉 score < 65 的LONG信号

elif dominant_side == 'SHORT':
    # 恐慌性杀跌 = 趋势确认 → 全部SHORT放行，不惩罚
    pass  # 不过滤
```

## 八、完整开仓校验流程（13道关卡）

```
01. [评级检查]  trading_symbol_rating.rating_level < 3
02. [V1评分]    计算14个组件得分 → long_score / short_score
03. [组件数量]  signal_components >= 2（position_mid需>=3）
04. [方向一致]  不含反向组件（bearish组件不进LONG路径）
05. [仓位矛盾]  LONG不含position_high；SHORT不含position_low
06. [阈值校验]  score >= dynamic_threshold（见第三节）
07. [上限拒绝]  score <= 150（超高分=追涨杀跌，历史胜率32.8%）
08. [V2冲突]    V2反向且强度>=20 → 拒绝（除非V1 score >= 阈值+25）
09. [Big4 veto] 强度>=40的逆势方向 → 硬封（仅过滤器开启时）
10. [黑名单]    signal_blacklist 组合匹配 → 拒绝
11. [评级限制]  rating_level=1/2 → 降低保证金至100U（不拒绝）
12. [羊群防御]  Big4=NEUTRAL + 80%同向 → LONG+5惩罚，SHORT全放
13. [开仓执行]  进入 KlinePullbackEntryExecutor 或 SmartEntryExecutor
```

## 九、权重自动优化机制

**文件**：`app/services/scoring_weight_optimizer.py`
**触发**：每日00:00 UTC，由 `app/daily_review_report.py` 调用

```
调整逻辑：
  性能分 > +10 → 权重 +3
  性能分 > +5  → 权重 +2
  性能分 < -5  → 权重 -2
  性能分 < -10 → 权重 -3
  其他         → 不变

权重范围：[5, 30]（强制上下限）

最低保护地板（防止核心空头信号被压至无效）：
  breakdown_short:      最低 25
  volume_power_bear:    最低 15
  trend_1h_bear:        最低 15
  volume_power_1h_bear: 最低 12
  consecutive_bear:     最低 12

性能分计算：
  win_rate_score = (胜率 - 0.50) × 100     # 50%胜率为基准
  pnl_score = avg_pnl / 5 × 100             # $5为基准
  performance_score = win_rate_score×0.6 + pnl_score×0.4

数据要求：单方向至少5笔订单才触发调整
```
