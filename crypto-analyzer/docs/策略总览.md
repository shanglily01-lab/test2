### 策略①：主策略 — V2信号评分趋势跟随

**文件**：`smart_trader_service.py`（U本位）/ `coin_futures_trader_service.py`（币本位）

**触发频率**：每5分钟扫描所有配置交易对

**核心逻辑**：

```
K线评分(14个信号组件) → Big4市场过滤 → 开仓阈值判断 → K线回调建仓
```

#### 信号评分

- **14个V1信号组件**，各自独立评分累加
- LONG满分：217分 | SHORT满分：243分
- **动态权重**：每日00:00 UTC自动优化（ScoringWeightOptimizer）
- 权重范围：[5, 30]，每次调整幅度±2~3

主要信号组件（做多方向）：
| 信号 | 说明 |
|------|------|
| breakout_long | 价格突破近期高点 |
| volume_power_bull | 放量上涨 |
| trend_1h_bull | 1H均线多头排列 |
| rsi_oversold | RSI超卖反弹 |
| macd_golden_cross | MACD金叉 |

#### Big4市场过滤

BTC(50%) + ETH(30%) + BNB(10%) + SOL(10%) 加权评分：

| Big4信号 | LONG门槛 | SHORT门槛 |

| BULLISH/STRONG_BULLISH | 60+拥挤惩罚 | 60 |
| NEUTRAL | 65 | 60 |
| BEARISH/STRONG_BEARISH | 100（封死） | 60 |

> Big4过滤器可通过 `system_settings.big4_filter_enabled` 开关（关闭后软调整模式）

#### TOP30过滤

- **实盘开启时**（`live_trading_enabled=1`）：只交易 `top_performing_symbols` TOP30交易对
- **模拟盘**：无限制，所有配置交易对均可开仓

#### 建仓方式（K线回调）

分3批建仓：30% → 30% → 40%，等待K线回调确认再入场，每批独立position记录

### 策略②：BTC动量跟随策略

**文件**：`app/services/btc_momentum_trader.py`

**触发频率**：每轮主循环（约每分钟）检测，内部4小时冷却

**核心思路**：BTC短期急涨急跌时，带动全市场同向运动，跟随做TOP30

#### 触发条件

监测4个时间窗口（15 / 30 / 45 / 60分钟），**任意一个**满足：

```
|BTC涨跌幅| ≥ 1.5%  →  触发
```

- 涨幅 ≥ 1.5% → 开 LONG
- 跌幅 ≥ -1.5% → 开 SHORT

#### 开仓规则

1. 获取 `top_performing_symbols` TOP30列表
2. 遍历每个交易对：
   - 已有同向仓 → 保留，跳过
   - 已有反向仓 → 先平仓，再开新仓
   - 无仓 → 直接开仓
3. 使用实时WS价格（fallback数据库最新K线）

#### 参数

| 参数 | 模拟盘 | 实盘 |
|------|-------|------|
| 保证金 | 400U | 账号配置（默认100U） |
| 杠杆 | 5x | 账号配置（默认5x） |
| 止损 | 2% | 2% |
| 止盈 | 6% | 6% |
| 冷却期 | 触发后4小时内不再触发 | 同 |

#### 实盘同步

`live_trading_enabled=1` 时，自动同步到 `user_api_keys` 中所有 `status='active'` 的账号

#### 价格历史

- 启动时自动从 `kline_data`（1M K线）预加载最近90分钟BTC价格，防止重启后等待
- 运行时每分钟追加，保留最近90分钟滚动窗口

### 策略③：市场预测策略（双周期共振）

**文件**：`app/services/market_predictor.py`

**触发频率**：每6小时运行一次（`app/main.py` 调度）

**核心思路**：独立于信号评分系统，纯粹基于技术指标形态判断未来3~4小时方向

#### 分析逻辑

**数据输入**：
- 1H K线：最近72根（3天）
- 15M K线：最近96根（24小时）

**1H指标**（大方向）：
| 指标 | 条件 |
|------|------|
| EMA9 vs EMA26 | EMA9 > EMA26 → 多头排列 |
| MACD柱(8/21/5) | 柱 > 0 且 最新 ≥ 上一根 → 多头扩张 |
| RSI(14) | 用于置信度评分 |
| ADX(14) | 用于趋势强度评分 |

**15M指标**（短期动能）：
| 指标 | 条件 |
|------|------|
| 近8根涨跌统计 | ≥5根上涨 → 15M多头 |
| RSI方向 | 近4根RSI上升 → 动能向上 |
| 成交量 | 近4根 > 前4根均量×1.3 → 量能放大 |

**方向判断**（双周期必须共振）：

```
1H多头 AND 15M多头  →  BULLISH（预测做多）
1H空头 AND 15M空头  →  BEARISH（预测做空）
其他               →  NEUTRAL（不开单）
```

#### 置信度评分（0~100分）

| 条件 | 加分 |
|------|------|
| 1H+15M方向一致（非NEUTRAL） | +30 |
| ADX > 30（强趋势） | +25 |
| ADX 20~30（弱趋势） | +10 |
| RSI < 40 或 > 60（动能明确） | +15 |
| MACD柱同向扩大 | +15 |
| 成交量放大 > 1.3x | +15 |

最高100分，**≥70分才开真实模拟单**

#### 输出：两套并行运行

**A. 虚拟回测单**（预测验证，不限数量）
- 存入 `prediction_backtest` 表
- **每个交易对都开，无数量限制**
- 100U × 5x，SL 2%，TP 6%
- 仅用于统计预测准确率，不影响实际资金

**B. 真实模拟单**（实际执行，最多5单）
- 存入 `futures_positions`（account_id=2，模拟盘）
- source = 'PREDICTOR'
- 条件：confidence ≥ 70，方向非NEUTRAL
- **最多同时持仓5单**（已有5单则本轮不开新单）
- 100U × 5x，SL 2%，TP 6%
- **6小时后自动平仓**（通过 `planned_close_time` 字段，由 SmartExitOptimizer 执行）
- **实盘同步上限同为5单**（如未来接入实盘同步，限制保持一致）

#### 自动平仓机制

1. **到期平仓**：开仓时写入 `planned_close_time = now + 6H`，SmartExitOptimizer 定期扫描并平仓
2. **止损平仓**：价格触及 entry_price × (1 - 2%)
3. **止盈平仓**：价格触及 entry_price × (1 + 6%)
4. **保底清理**：market_predictor 每次运行时额外检查，平掉持仓 > 5.5H 的 PREDICTOR 单
