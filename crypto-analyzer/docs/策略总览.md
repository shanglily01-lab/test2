> 更新时间：2026-03-25

---

## 一、系统整体架构

```
Binance 实时价格 (WebSocket)
        │
        ▼
┌──────────────────────────────────────────────────────┐
│                  策略层（并行运行）                     │
│                                                      │
│  ① 主策略 V1信号评分       每5分钟扫描全交易对           │
│  ② BTC动量跟随策略         每分钟检测BTC价格变动         │
│  ③ 市场预测策略            每6小时分析+开单              │
│  ④ 15M突破策略             每15分钟扫描                  │
│  ⑤ 币本位合约策略          独立进程，独立逻辑             │
└──────────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────────┐
│                    平仓层                              │
│  SmartExitOptimizer                                  │
│  — 止损 / 止盈 / 最大持仓时间 / planned_close_time      │
└──────────────────────────────────────────────────────┘
        │
        ▼
  futures_positions       （模拟盘 account_id=2）
  live_futures_positions  （实盘，user_api_keys 中活跃账号）
```

## 三、策略详情

---

### 策略①：主策略 V1 信号评分

**文件**：`smart_trader_service.py`

**触发频率**：每5分钟扫描所有配置交易对（约300个）

**核心逻辑**：

```
K线数据 → 18个V1组件计分 → Big4动态阈值过滤 → V2协同确认 → K线回调建仓
```

#### 信号组件（18个，满分 LONG=244 / SHORT=270）

| 分类 | 组件 | LONG | SHORT | 说明 |
|------|------|------|-------|------|
| 位置(100H窗口) | position_low | +20 | — | 价格在100H区间底部30%以内 |
| 位置(100H窗口) | position_high | — | +20 | 价格在100H区间顶部70%以上 |
| 位置(100H窗口) | position_mid | +5 | +5 | 中间区间，顺量能方向加分 |
| 位置(24H窗口) | position_24h_low | +12 | — | 当日低位，更敏感，仿币本位 |
| 位置(24H窗口) | position_24h_high | — | +12 | 当日高位，Big4非强牛时触发 |
| 动量 | momentum_up_3pct | +15 | — | 24H涨幅 > 3% |
| 动量 | momentum_down_3pct | — | +15 | 24H跌幅 > 3% |
| 趋势 | trend_1h_bull | +20 | — | 24根1H中阳线≥15根 |
| 趋势 | trend_1h_bear | — | +20 | 24根1H中阴线≥14根 |
| 波动率 | volatility_high | +10 | +10 | 24H波幅>5%，顺主方向加分 |
| 连续K线 | consecutive_bull | +15 | — | 近10H阳线≥7根且涨幅<5% |
| 连续K线 | consecutive_bear | — | +15 | 近10H阴线≥7根且跌幅>-10% |
| 量能1.5×双框 | volume_power_bull | +25 | — | 1H+15M净力量≥2（阈值1.5×均量） |
| 量能1.5×双框 | volume_power_bear | — | +25 | 1H+15M净力量≤-2 |
| 量能1.5×单框 | volume_power_1h_bull | +15 | — | 仅1H净力量≥3 |
| 量能1.5×单框 | volume_power_1h_bear | — | +15 | 仅1H净力量≤-3 |
| 量能1.2×双框 | volume_power_12x_bull | +15 | — | 1H+15M净力量≥2（阈值1.2×，更敏感） |
| 量能1.2×双框 | volume_power_12x_bear | — | +15 | 1H+15M净力量≤-2（阈值1.2×） |
| 破位追空 | breakdown_short | — | +20 | 低位(<30%)+1H+15M双框强空 |
| Big4顺势 | big4_strong_bull_cont | +15 | — | Big4 STRONG_BULLISH + 自身涨>2% |

> 净力量 = 放量阳线数 - 放量阴线数（放量 = 成交量 > N × 24H均量）

#### 币本位风格补充层（2026-03-25 新增）

原始 U 本位使用 **100H 位置窗口** + **1.5× 量能阈值**，在震荡行情中不易触发信号。
币本位策略发现用 **24H 短窗口 + 1.2× 宽松阈值** 对当日走势更敏感，效果更好，因此将同套逻辑作为补充层移植到 U 本位：

| 新增组件 | 对应原有组件 | 区别 |
|---------|------------|------|
| position_24h_low | position_low | 位置窗口 24H（原 100H），对当日极值更敏感 |
| position_24h_high | position_high | 同上 |
| volume_power_12x_bull | volume_power_bull | 放量阈值 1.2×（原 1.5×），更易触发 |
| volume_power_12x_bear | volume_power_bear | 同上 |

两套组件并存：原有组件（100H/1.5×）保持不变，新组件作为额外加分项。
权重由 ScoringWeightOptimizer 每日自动调优，初始权重：position_24h ±12，volume_12x ±15。

#### Big4 动态开仓阈值

| Big4状态（强度≥65） | LONG阈值 | SHORT阈值 |
|-------------------|---------|---------|
| BULLISH / STRONG_BULLISH | 60 | 60 |
| NEUTRAL | 65 | 60 |
| BEARISH / STRONG_BEARISH | 100（封死） | 60 |

附加条件：
- 至少2个信号组合，方向一致（无反向信号）
- V2协同过滤（coin_kline_scores 表）：V2反向时需额外+25分才放行
- 信号黑名单检查（signal_blacklist 表）

#### 建仓方式（K线回调）

分3批建仓：30% → 30% → 40%，等K线回调确认再入场

#### 权重自动优化

每日 00:00 UTC，ScoringWeightOptimizer 根据近7日胜率自动调整各组件权重，范围 [5, 30]，每次幅度±2~3

### 策略②：BTC 动量跟随

**文件**：`app/services/btc_momentum_trader.py`

**触发频率**：每分钟检测 BTC 价格，内部4小时冷却

**核心思路**：BTC 短期急涨急跌带动全市场，跟随做 TOP30

#### 触发条件

监测4个时间窗口（15 / 30 / 45 / 60 分钟），**任意一个**满足：

```
|BTC涨跌幅| ≥ 1.5%  →  触发
涨幅 > 0  →  开 LONG
跌幅 < 0  →  开 SHORT
```

#### 开仓规则

1. 获取 `top_performing_symbols` TOP30 列表
2. 对每个交易对：
   - 已有同向仓 → 保留，跳过
   - 已有反向仓 → 先平仓，再开新仓
   - 无仓 → 直接开仓
3. 冷却：触发后 4 小时内不再触发

#### 参数

| 参数 | 模拟盘 | 实盘 |
|------|-------|------|
| 保证金 | 400U/单 | max_position_value（默认100U） |
| 杠杆 | 5x | max_leverage（默认5x） |
| 止损 | 2% | 2% |
| 止盈 | 6% | 6% |

#### 实盘同步

`live_trading_enabled=1` 时，对所有 `user_api_keys.status='active'` 账号调用 BinanceFuturesEngine 真实下单，成功后发 TG 推送

### 策略③：市场预测器

**文件**：`app/services/market_predictor.py`

**触发频率**：每6小时（app/main.py 调度，00:00 / 06:00 / 12:00 / 18:00 UTC）

**核心思路**：独立于信号评分体系，纯粹基于技术指标形态判断未来6小时方向

#### 分析逻辑

**数据**：72根1H K线 + 96根15M K线

**1H 指标（大方向）**：

| 指标 | 多头条件 |
|------|---------|
| EMA9 vs EMA26 | EMA9 > EMA26 |
| MACD(8/21/5) | 柱 > 0 且 ≥ 上一根（扩张中） |
| RSI(14) | 用于置信度 |
| ADX(14) | 用于趋势强度 |

**15M 指标（短期动能）**：

| 指标 | 多头条件 |
|------|---------|
| 近8根方向 | ≥5根上涨 |
| RSI趋势 | 近4根RSI持续上升 |
| 成交量 | 近4根 > 前4根均量×1.3 |

**方向判断（双周期共振）**：

```
1H 多头 AND 15M 多头  →  BULLISH
1H 空头 AND 15M 空头  →  BEARISH
其他                  →  NEUTRAL（不开单）
```

#### 置信度评分（0~100分）

| 条件 | 加分 |
|------|------|
| 1H+15M 方向一致（非NEUTRAL） | +30 |
| ADX > 30（强趋势） | +25 |
| ADX 20~30（弱趋势） | +10 |
| RSI < 40 或 > 60（动能明确） | +15 |
| MACD 柱同向扩大 | +15 |
| 成交量放大 > 1.3x | +15 |

#### 两套并行输出

**A. 虚拟回测单**（`prediction_backtest` 表，不限数量）
- confidence ≥ 40，每个交易对独立开，仅统计预测精度，不影响实际资金

**B. 真实模拟单**（`futures_positions`，最多5单）
- confidence ≥ 70，source = 'PREDICTOR'
- 保证金100U × 5x，SL 2%，TP 6%
- 开仓时写入 `planned_close_time = NOW() + 6H`，6小时后自动平仓

#### 平仓机制

1. **到期平仓**：`planned_close_time` 到期 → SmartExitOptimizer 执行（同步平实盘）
2. **保底清理**：每次运行时额外平掉持仓 > 5.5H 的 PREDICTOR 单
3. **止损/止盈**：SmartExitOptimizer 监控

#### 实盘同步

`live_trading_enabled=1` 时，confidence ≥ 70 的单同步到实盘（上限同为5单），成功后发 TG 推送

### 策略④：15M 突破策略

**文件**：`app/services/breakout_15m_trader.py`

**触发频率**：每15分钟扫描全部配置交易对

**核心思路**：捕捉震荡市中 15M 级别的量价突破信号，趋势信号未反转时提前入场

#### 三个条件同时满足才入场

| 条件 | 做多 | 做空 |
|------|------|------|
| 价格突破 | 当前价 > 前8根15M最高价 | 当前价 < 前8根15M最低价 |
| RSI 拐头 | RSI(14)>50 且较4根前上升≥3点 | RSI(14)<50 且较4根前下降≥3点 |
| 量能放大 | 最新1根量 > 前8根均量 × 1.5 | 同上 |

#### 参数

| 参数 | 值 |
|------|-----|
| 保证金 | 100U/单 |
| 杠杆 | 5x |
| 止损 | 2% |
| 止盈 | 5% |
| 同币种同向冷却 | 4小时 |
| 最大持仓 | 10单 |

#### 实盘同步

`live_trading_enabled=1` 时，开仓后调用 BinanceFuturesEngine 同步实盘（绑定 paper_position_id），成功后发 TG 推送

### 策略⑤：币本位合约策略

**文件**：`coin_futures_trader_service.py`

**账户**：account_id = 3，交易对格式 BTC/USD

**逻辑**：独立评分体系（9个组件，与U本位不共享权重），详见 [币本位超级大脑逻辑梳理.md](币本位超级大脑逻辑梳理.md)

| 参数对比 | U本位主策略 | 币本位 |
|---------|-----------|-------|
| 止损 | 2% | 3% |
| 止盈 | 6% | 2% |
| 位置窗口 | 100H | 24H |
| 量能阈值 | 1.5× | 1.2× |

## 四、策略适用场景对比

| 策略 | 最适合 | 最弱场景 |
|------|--------|---------|
| 主策略 V1 | 中长趋势明确，Big4 单边 | 震荡横盘（评分积累慢） |
| BTC 动量 | BTC 急速单边行情 | 宽幅震荡（反复触发） |
| 预测器 | 趋势启动初期 | 高波动快速反转 |
| 15M 突破 | 震荡后放量突破 | 趋势市（易假突破频繁触发） |
| 币本位 | 短线震荡波动 | 长周期趋势（止盈2%易被扫损） |

四个U本位策略互补：趋势市靠主策略+BTC动量，震荡市靠预测器+15M突破

## 五、平仓层：SmartExitOptimizer

**文件**：`app/services/smart_exit_optimizer.py`

统一管理所有策略的模拟盘持仓，触发条件：

| 触发条件 | 说明 |
|---------|------|
| 止损 | mark_price 触及 stop_loss_price |
| 止盈 | mark_price 触及 take_profit_price |
| 最大持仓时间 | 超过 max_hold_hours（system_settings） |
| 计划平仓时间 | 超过 planned_close_time（预测器专用） |

平仓时同步检查 `live_futures_positions.paper_position_id`，有对应实盘单则同步调用交易引擎平仓

## 六、自学习优化循环

```
交易结果 (futures_positions.realized_pnl)
    │
    ▼
daily_review_report.py（每日 00:00 UTC）
    │
    ├── 发 Telegram 复盘报告（4条）
    │
    └── ScoringWeightOptimizer.adjust_weights()
            │
            ├── 读取近7天已平仓记录
            ├── 按 Big4 市场状态过滤（熊市多头亏损不污染多头权重）
            ├── 计算各组件胜率 + 平均盈亏
            └── 调整 signal_scoring_weights 表
                    权重范围 [5, 30]，每次 ±2~3
```

## 九、数据流简图

```
K线采集（每分钟）
    └→ kline_data 表
            │
            ├→ coin_kline_scores（每5分钟评分）
            │       └→ 主策略① 读取V1+V2信号
            │
            ├→ btc_momentum_trader（每分钟）
            │       └→ BTC动量策略②
            │
            ├→ market_predictor（每6小时）
            │       └→ 预测策略③ → prediction_backtest + futures_positions
            │
            └→ breakout_15m_trader（每15分钟）
                    └→ 15M突破策略④

开仓记录
    ├→ futures_positions（模拟盘 account_id=2）
    └→ live_futures_positions（实盘，paper_position_id 关联模拟单）

平仓监控
    └→ SmartExitOptimizer（持续运行，平仓时同步实盘）

每日自优化
    └→ daily_review_report.py → signal_scoring_weights
```
