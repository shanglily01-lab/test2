## 📌 重要策略调整记录 (2026-01-12)

### 问题发现
通过7天交易数据分析（2026-01-05 至 2026-01-12）发现严重问题：
- **总盈亏**: -$1024
- **胜率**: 仅24% (12胜/38败)
- **快速止损**: 26笔交易在2小时内被止损
- **核心问题**: 信号强度0.15% < 回调幅度0.8%-1.8%

### 根本原因
**限价单在假突破后的真反转时成交**：
- 15M周期EMA差距仅0.15%（噪音级别，非真趋势）
- 限价单设置market_minus 0.8%-1.8%（等回调入场）
- 当价格回调触发限价单时，0.15%的"趋势"已被0.8%的回调打破
- 结果：买在假突破后的真下跌起点

**典型案例** (DOGE/USDT):
```
时间线：
T0: EMA9上穿EMA26 0.15% → 系统设置限价单 market-0.8%
T1: 价格回调0.8%，限价单成交 → 此时趋势已反转！
T2: 价格继续下跌
T3: 30分钟后触发硬止损 -18.25%
```

### 解决方案
**提高最小信号强度，确保 趋势强度 >> 回调幅度**（安全比例 ≥ 1.5倍）：

| 策略 | 旧信号强度 | 新信号强度 | 回调幅度 | 安全比例 |
|------|-----------|-----------|---------|---------|
| level-1 | 0.15% | **1.2%** | 0.8% | 1.5x ✓ |
| level2  | 0.15% | **1.5%** | 1.0% | 1.5x ✓ |
| level-3 | 0.15% | **2.0%** | 1.2% | 1.67x ✓ |
| level4  | 0.15% | **2.5%** | 1.4% | 1.79x ✓ |

### 预期效果
1. **大幅减少假信号**: 只在真正强趋势时开仓
2. **降低快速止损率**: 预期从52%降至<15%
3. **提高胜率**: 预期从24%提升至40-50%
4. **改善盈亏**: 预期实现正收益或大幅减亏

### 做空策略建议
根据数据分析，做空表现极差：
- **做多**: 130笔，胜率53.8%，亏损-$232
- **做空**: 51笔，胜率31.4%，亏损-$1056 ❌

**建议**: 暂时禁用做空，专注做多；或提高做空信号强度至≥2.5%

## 一、开仓信号

所有开仓信号都需要**双周期确认**：15M信号 + 1H趋势方向一致才能开仓。

### 1. EMA金叉/死叉信号 (ema_cross)

**做多条件**：
- 15M周期：EMA9上穿EMA26（金叉）
- 1H周期：EMA9 > EMA26（多头趋势）

**做空条件**：
- 15M周期：EMA9下穿EMA26（死叉）
- 1H周期：EMA9 < EMA26（空头趋势）

**信号类型**: `ema_cross`

---

### 2. 持续趋势入场 (sustained_trend_entry)

错过金叉/死叉后，在趋势持续时仍可入场。

**做多条件**：
- 15M周期：EMA9 > EMA26
- 1H周期：EMA9 > EMA26（多头确认）
- 15M趋势强度：0.15% ≤ EMA差值百分比 ≤ 1.0%
- MA10/EMA10确认：EMA10 > MA10
- 价格确认：当前价 > EMA9
- 冷却时间：60分钟

**做空条件**：
- 15M周期：EMA9 < EMA26
- 1H周期：EMA9 < EMA26（空头确认）
- 15M趋势强度：0.15% ≤ EMA差值百分比 ≤ 1.0%
- MA10/EMA10确认：EMA10 < MA10
- 价格确认：当前价 < EMA9
- 冷却时间：60分钟

**信号类型**: `sustained_trend_entry`

---

### 3. 连续趋势信号 (sustained_trend)

检测15M和5M周期EMA差值同时放大。

**做多条件**：
- 15M周期：EMA9 > EMA26
- 5M周期：EMA9 > EMA26，且差值放大
- 1H周期：EMA9 > EMA26（方向确认）

**做空条件**：
- 15M周期：EMA9 < EMA26
- 5M周期：EMA9 < EMA26，且差值放大
- 1H周期：EMA9 < EMA26（方向确认）

**信号类型**: `sustained_trend`

---

### 4. 震荡反向信号 (oscillation_reversal)

检测震荡区间内的反转信号（连续同向K线+成交量变化）。

**做多条件**：
- 15M周期：连续4根阴线
- 幅度：4根K线波动 < 0.5%
- 成交量：放量（≥ 1.5倍前4根均值）
- 1H周期：EMA9 > EMA26（多头确认）

**做空条件**：
- 15M周期：连续4根阳线
- 幅度：4根K线波动 < 0.5%
- 成交量：缩量（≤ 0.7倍前4根均值）
- 1H周期：EMA9 < EMA26（空头确认）

**信号类型**: `oscillation_reversal`

---

### 5. RSI超买超卖信号 (rsi_extremes) ⚡ 新增

检测RSI指标的极值情况，捕捉超跌反弹和超涨回落机会。

**做多条件** (RSI超卖反弹):
- 15M周期：RSI < 20（超卖）
- 1H周期：EMA9 > EMA26（多头趋势确认）
- 限价单：建议market_minus_0.8%（等小幅回调入场）

**做空条件** (RSI超买回落):
- 15M周期：RSI > 80（超买）
- 1H周期：EMA9 < EMA26（空头趋势确认）
- 限价单：建议market_plus_0.8%（等小幅反弹入场）

**信号类型**: `rsi_extremes`

**使用建议**:
- RSI极值出现频率低，需耐心等待
- 必须配合1H趋势确认，避免逆势交易
- 超卖做多胜率高于超买做空
- 适合捕捉短期反转机会

**典型案例** (BTC/USDT RSI超卖做多):
```
T0: 15M RSI跌至18.5（超卖）
T1: 1H EMA9 > EMA26（多头趋势确认）✓
T2: 系统触发RSI超卖做多信号
T3: 挂限价单 market_minus_0.8% = $95,200
T4: 价格反弹至$95,200，限价单成交
T5: 快速反弹至$96,500，触发移动止盈
结果: 盈利+1.37%，持仓时间45分钟
```

## 二、平仓逻辑

### 1. 硬止损 (hard_stop_loss)

固定止损，达到亏损百分比立即平仓。

**触发条件**：
- 做多：当前价 ≤ 入场价 × (1 - 止损%)
- 做空：当前价 ≥ 入场价 × (1 + 止损%)

**默认值**：2.5%（可配置 `stopLoss`）

**冷却时间**：无（立即触发）

**平仓代码**: `hard_stop_loss`

---

### 2. 最大止盈 (max_take_profit)

固定止盈，达到盈利目标立即平仓。

**触发条件**：
- 盈利百分比 ≥ 止盈目标

**默认值**：3.0%（可配置 `takeProfit`）

**平仓代码**: `max_take_profit|pnl:X.XX%`

---

### 3. 移动止盈 (trailing_take_profit)

跟踪最高盈利，回撤一定比例时平仓，锁定利润。

**触发条件**：
1. **激活条件**：最高盈利 ≥ 激活阈值
2. **平仓条件**：回撤百分比 ≥ 回撤阈值

**配置参数**：
- `trailingActivate`: 激活阈值（默认1.5%）
- `trailingCallback`: 回撤阈值（默认0.5%）

**计算公式**：
- 回撤% = 最高盈利% - 当前盈利%

**最小持仓时间**：15分钟（避免刚开仓就被平）

**冷却时间**：15分钟（可配置 `trailingCooldownMinutes`）

**平仓代码**: `trailing_take_profit|max:X.XX%|cb:X.XX%`

**示例**：
- 入场后盈利达到2.1%，移动止盈激活
- 价格回落，当前盈利1.6%
- 回撤 = 2.1% - 1.6% = 0.5% ≥ 回撤阈值0.5%
- 触发平仓，实际盈利1.6%

---

### 4. EMA差值收窄止盈 (ema_diff_narrowing_tp)

趋势减弱时止盈，EMA9和EMA26距离收窄。

**触发条件**：
- 当前EMA差值百分比 < 0.5%
- 当前盈利 ≥ 1.5%

**配置参数**：
- `emaDiffTakeProfit.threshold`: 差值阈值（默认0.5%）
- `emaDiffTakeProfit.minProfitPct`: 最小盈利（默认1.5%）

**平仓代码**: `ema_diff_narrowing_tp|diff:X.XX%|pnl:X.XX%`

---

### 5. 趋势减弱平仓 (trend_weakening)

开仓后30分钟，EMA差值连续3次减弱时平仓。

**触发条件**：
- 持仓时间 ≥ 30分钟
- 当前盈利 ≥ 1.0%
- EMA差值百分比 < 0.5%

**配置参数**：
- `trendWeakeningExit.enabled`: 是否启用（默认true）
- `trendWeakeningExit.emaDiffThreshold`: 差值阈值（默认0.5%）
- `trendWeakeningExit.minProfitPct`: 最小盈利（默认1.0%）

**平仓代码**: `trend_weakening|ema_diff:X.XX%|pnl:X.XX%`

---

### 6. 信号反转平仓

#### 6.1 金叉/死叉反转 (ema_direction_reversal_tp)

15M周期EMA交叉反向时平仓。

**触发条件**：
- 做多持仓：15M出现死叉（EMA9下穿EMA26）
- 做空持仓：15M出现金叉（EMA9上穿EMA26）
- 当前盈利 ≥ 1.0%

**最小盈利要求**：1.0%

**平仓代码**: `death_cross_reversal|pnl:X.XX%` 或 `golden_cross_reversal|pnl:X.XX%`

## 三、移动止损逻辑 (trailing_stop_loss)

动态调整止损价格，保护浮盈。

**激活条件**：
- 当前盈利 ≥ 激活阈值

**止损价格计算**：
- 做多：止损价 = 当前价 × (1 - 距离%)
- 做空：止损价 = 当前价 × (1 + 距离%)

**入场价保护**：
- 做多：止损价不能 ≥ 入场价
- 做空：止损价不能 ≤ 入场价

**最小移动阈值**：0.1%（避免频繁微调）

**配置参数**：
- `smartStopLoss.trailingStopLoss.enabled`: 是否启用
- `smartStopLoss.trailingStopLoss.activatePct`: 激活阈值（默认1.0%）
- `smartStopLoss.trailingStopLoss.distancePct`: 距离百分比（默认1.0%）

**冷却时间**：15分钟（可配置 `trailingCooldownMinutes`）

**示例**：
- 入场价 $100，当前价 $102（盈利2%）
- 激活阈值1%，距离1%
- 止损价 = $102 × (1 - 1%) = $100.98
- 价格继续上涨到$103，止损价更新为 $103 × 0.99 = $101.97

## 四、紧急停止机制 (Circuit Breaker)

当短时间内发生多次硬止损时，自动暂停所有交易并平掉所有持仓，防止系统性风险导致重大亏损。

### 1. 触发条件

系统每次平仓后都会自动检查是否触发紧急停止：

**检查范围**：最近5笔平仓记录

**硬止损判断标准**：
- `unrealized_pnl_pct <= -10%`（保证金亏损达到10%或更多）
- 对应约2%的价格亏损（使用5倍杠杆时）

**触发阈值**：
- 最近5笔中有 **3笔或更多** 是硬止损 → 触发紧急停止

**实现位置**：[circuit_breaker.py](app/services/circuit_breaker.py)

---

### 2. 执行动作

紧急停止触发后，系统会立即执行以下操作：

#### 2.1 暂停所有策略
```sql
UPDATE trading_strategies SET enabled = 0 WHERE enabled = 1
```
- 包括模拟盘和实盘策略
- 所有自动开仓信号将被忽略

#### 2.2 平掉所有持仓

**平仓方式**：同步顺序执行（非并发）
- 遍历所有未平仓持仓
- 每个持仓单独try-except处理
- 使用市价单立即平仓
- 平仓原因记录为：`emergency_stop`

**关键实现细节**：
```python
for position in positions:
    try:
        result = futures_engine.close_position(
            position_id=position['id'],
            close_reason="emergency_stop"
        )
        if result and result.get('success'):
            success_count += 1
        else:
            failed_positions.append(position)
    except Exception as e:
        failed_positions.append(position)
        logger.error(f"平仓异常: {position['symbol']}")
```

**失败处理**：
- 失败的持仓会记录到`failed_positions`列表
- 使用`logger.critical`级别记录失败详情
- 不会因为单个持仓失败而中断整个流程

#### 2.3 冷却期设置

**冷却时长**：4小时

**冷却期间**：
- 所有策略保持暂停状态（`enabled = 0`）
- 不接受新的开仓信号
- 可以手动开仓（但不推荐）

**自动恢复**：
- 4小时后系统会自动恢复所有策略（`enabled = 1`）

### 3. 检测逻辑演变

#### 早期版本（已废弃）
```python
# 通过notes字段文本匹配 - 已弃用
if 'hard_stop_loss' in notes:
    hard_stop_count += 1
```

**问题**：
- notes字段格式不统一（有英文`'hard_stop_loss'`也有中文`'硬止损平仓(亏损3.02% >= 3%)'`）
- 导致中文格式的硬止损未被识别
- **实际案例**：2026-01-06晚，第3笔硬止损时未触发紧急停止，最终18笔硬止损才停止，亏损-1011 USDT

```python
# 基于实际盈亏百分比判断 - 当前使用

```

**优点**：
- 不依赖文本格式，完全基于数值判断
- 准确识别所有硬止损（无论notes如何记录）
- 阈值从-12.5%调整为-10%，提高灵敏度

### 8. 重要注意事项

1. **平仓是同步执行**：`close_position()`是同步函数，不能用`asyncio.gather()`并发执行
2. **硬止损永远优先**：即使在冷却期内，硬止损（-2.5%）也会立即触发平仓
3. **百分比基于保证金**：`unrealized_pnl_pct`是基于保证金的盈亏，不是价格涨跌幅
4. **实盘同步**：如果策略配置了`syncLive=1`，紧急停止会同时平掉实盘持仓
5. **失败容错**：单个持仓平仓失败不会中断整个流程，会继续处理其他持仓

## 六、平仓优先级

系统按以下顺序检查平仓条件：

1. **硬止损** - 最高优先级，防止大额亏损
2. **移动止损** - 保护浮盈
3. **最大止盈** - 固定目标
4. **移动止盈** - 跟踪最高点回撤
5. **趋势减弱** - 趋势弱化时退出
6. **EMA收窄止盈** - 趋势即将反转
7. **信号反转** - 交叉信号反向

每个平仓条件都有独立的冷却时间和最小盈利要求，避免过早平仓。

- ✅ 止盈平仓执行时检查是否变成亏损，如果是则取消平仓

## 八、冷却时间机制

系统设置了多种冷却时间，避免频繁交易和过早平仓。

### 1. 开仓冷却 (Entry Cooldown)

**配置参数**：`entryCooldown`

**作用**：平仓后一段时间内禁止该方向再次开仓。

**限制条件**：
- 每个币种、每个方向最多同时持有N个仓位（`maxPositionsPerDirection`，默认1个）
- 平仓后冷却时间内不能再次开仓同方向

**配置示例**：
```json
{
  "entryCooldown": {
    "enabled": true,
    "minutes": 30,
    "maxPositionsPerDirection": 1
  }
}
```

**默认值**：
- `enabled`: true（启用）
- `minutes`: 30分钟
- `maxPositionsPerDirection`: 1个

**触发场景**：
- BTC/USDT 做多持仓平仓后
- 30分钟内不能再次开BTC/USDT做多
- 但可以开BTC/USDT做空（方向不同）

---

### 2. 平仓冷却 (Close-Reopen Cooldown)

**配置参数**：`closeReopenCooldownMinutes`

**作用**：平仓后短时间内禁止同方向重新开仓（与开仓冷却类似但更严格）。

**默认值**：15分钟

**特殊规则**：
- 如果配置了`closeReopenSameDirectionOnly=true`，只限制同方向
- 否则双向都限制

---

### 3. 反转预警冷却 (Reversal Warning Cooldown)

**配置参数**：`reversalWarning.cooldownMinutes`

**作用**：检测到趋势即将反转时，进入冷却期暂停该方向开仓，直到出现明确的交叉信号才解除。

**触发条件**：
- EMA斜率变化 > 阈值（`slopeChangeThreshold`，默认0.3）
- 或 EMA差值收缩 > 阈值（`diffShrinkThreshold`，默认30%）

**冷却解除条件**：
- 做多方向：出现明确金叉（EMA9上穿EMA26）
- 做空方向：出现明确死叉（EMA9下穿EMA26）
- 或超时（30分钟）

**默认值**：30分钟

**配置示例**：
```json
{
  "reversalWarning": {
    "enabled": true,
    "slopeChangeThreshold": 0.3,
    "diffShrinkThreshold": 30,
    "cooldownMinutes": 30
  }
}
```

---

### 4. 移动止盈/止损冷却 (Trailing Cooldown)

**配置参数**：`trailingCooldownMinutes`

**作用**：开仓后的保护期，避免刚开仓就被移动止盈/止损/反转信号平掉。

**保护机制**：
- **冷却期内**（开仓后15分钟）：
  - 跳过移动止盈检查
  - 跳过普通止损价触发（仅-1.93%的止损价）
  - 跳过信号反转平仓
  - 跳过5M信号止损
  - **硬止损（-2.5%）不受限制**，作为紧急止损

- **冷却期后**：
  - 所有平仓条件正常检查

**默认值**：15分钟

**特殊说明**：
- 移动止盈的`最小持仓时间`也是15分钟，与此冷却时间一致
- 硬止损永远不受冷却时间限制

---

### 5. 持续趋势入场冷却 (Sustained Trend Cooldown)

**配置参数**：`sustainedTrend.cooldownMinutes`

**作用**：持续趋势入场信号的冷却时间，避免在同一趋势中重复开仓。

**默认值**：60分钟

**说明**：
- 持续趋势入场成功后
- 60分钟内不再检查该币种该方向的持续趋势信号

---

### 6. 限价单冷却 (Limit Order Cooldown)

**作用**：限价单超时或取消后的冷却期。

**冷却时间**：30分钟（硬编码）

**触发条件**：
- 最近30分钟内有CANCELED或超时的限价单
- 同币种同方向同策略

**说明**：
- 防止限价单反复挂单、取消
- 给价格更多时间稳定

---

## 九、冷却时间优先级

当多个冷却同时存在时，检查顺序：

1. **反转预警冷却** - 最优先，趋势即将反转时禁止开仓
2. **平仓冷却** - 刚平仓后的冷却
3. **开仓冷却** - 普通开仓限制
4. **限价单冷却** - 限价单相关冷却

对于持仓的平仓检查：
1. **硬止损** - 永远不受冷却限制
2. **移动止盈/止损冷却** - 开仓后15分钟内保护
3. **其他平仓条件** - 冷却期后正常检查

## 十、实盘同步

模拟盘开仓后，可选同步到实盘（`syncLive=1`）。

### 开仓同步

**同步内容**：
- 开仓：实盘保证金按配置计算（固定金额或百分比）
- 杠杆倍数：与模拟盘相同
- 止损价格：同步硬止损百分比（`stopLoss`，默认2.5%）
- 止盈价格：同步最大止盈百分比（`takeProfit`，默认3.0%）

**止损止盈执行**：
- 由币安交易所自动监控执行
- 触发后自动平仓，无需程序干预

### 平仓同步

**同步机制**：
- ✅ **所有模拟盘平仓信号都会同步到实盘**
- 包括：硬止损、移动止盈、趋势减弱、EMA收窄、信号反转等
- 模拟盘平仓后，实盘立即执行全部平仓

**同步逻辑**：
```
模拟盘触发平仓 → 检查策略syncLive配置 → 同步实盘平仓
```

**特殊说明**：
- 移动止损/移动止盈虽然在模拟盘是动态调整，但**触发平仓时会同步实盘**
- 实盘平仓全部使用市价单，确保立即成交
- 如果实盘已被币安止损止盈自动平仓，同步操作会跳过

### 配置参数

- `syncLive`: 是否同步实盘（0=否，1=是）
- `liveQuantityPct`: 实盘仓位比例（已废弃，现在用固定保证金）

## 十一、限价单入场策略

### 概述

限价单策略使用**回调入场**的方式，在强趋势中等待价格回调到更优价格时买入/卖出。

**核心逻辑**: 强趋势 + 等回调 = 更优入场价格

### 双周期确认机制

限价单需要更强的趋势确认：
- **15M周期**: 用于计算趋势强度（信号强度）
- **1H周期**: 用于确认趋势方向

只有当15M和1H方向一致时，才会创建限价单。

### 最小信号强度要求 ⭐

**关键参数**: `minSignalStrength.ema9_26`

这是限价单策略最重要的参数，决定入场质量：

| 策略等级 | 信号强度 | 回调幅度 | 安全比例 | 说明 |
|---------|---------|---------|---------|------|
| level-1 | 1.2% | 0.8% (market_minus_0_8) | 1.5x | 最宽松 |
| level2  | 1.5% | 1.0% (market_minus_1) | 1.5x | 中等 |
| level-3 | 2.0% | 1.2% (market_minus_1_2) | 1.67x | 严格 |
| level4  | 2.5% | 1.4% (market_minus_1_4) | 1.79x | 最严格 |

**安全比例原则**: 信号强度应 ≥ 回调幅度 × 1.5倍

**为什么这么重要？**

如果信号强度 < 回调幅度，会导致：
1. 限价单在假突破后成交（买在反转起点）
2. 趋势已被回调打破，价格继续反向
3. 快速触发止损，大额亏损

**错误案例** (2026-01-06, 信号强度0.15%时):
```
DOGE/USDT:
- EMA差距: 0.15% (假突破)
- 限价单: market_minus_0.8% = $0.1509
- 结果: 限价单成交后，趋势已反转
- 30分钟后硬止损 -18.25%
```

**正确示例** (信号强度1.2%后):
```
BTC/USDT:
- EMA差距: 1.5% (真趋势)
- 限价单: market_minus_0.8% = $94,500
- 结果: 回调后趋势继续，盈利
```

### 限价单价格类型

**做多** (`longPrice`):
- `market`: 市价单（不使用限价）
- `market_minus_X`: 市价 - X%（等回调买入）
  - 可选: 0.8%, 1%, 1.2%, 1.4%, 1.6%, 1.8%, 2%, 3%, 4%, 5%, 6%, 8%

**做空** (`shortPrice`):
- `market`: 市价单（不使用限价）
- `market_plus_X`: 市价 + X%（等反弹卖出）
  - 可选: 0.8%, 1%, 1.2%, 1.4%, 1.6%, 1.8%, 2%, 3%, 4%, 5%, 6%, 8%

### 限价单创建条件

创建限价单需满足所有条件：

1. **策略启用限价单**: `enableLimitOrder = true`
2. **配置了限价价格**: `longPrice` 或 `shortPrice` ≠ `market`
3. **15M信号强度足够**: `ema_diff_pct_15m ≥ minSignalStrength.ema9_26`
4. **1H方向确认**: 15M和1H的EMA方向一致
5. **无重复持仓**: 该方向没有已开仓位
6. **无待执行限价单**: 没有PENDING状态的限价单
7. **不在冷却期**: 30分钟内没有超时/取消的限价单

### 限价单超时与取消

**有效期**: 2小时

**超时后**:
- 订单状态变为EXPIRED
- 进入30分钟冷却期
- 冷却期后重新检测信号

**手动取消后**:
- 订单状态变为CANCELLED
- 同样进入30分钟冷却期

### 最佳实践建议

1. **优先提高信号强度**，而非减小回调幅度
   - ✅ 信号强度1.5% + 回调1.0% = 安全
   - ❌ 信号强度0.5% + 回调0.3% = 仍有风险

2. **不同币种用不同等级**
   - 主流币(BTC/ETH): level-1 (回调小)
   - 中盘币: level2/level-3 (回调适中)
   - 小盘币: level4 (回调大，但信号强度要求高)

3. **定期复盘**
   - 使用 `analyze_recent_trades.py` 查看表现
   - 快速止损率 > 30% → 提高信号强度
   - 开仓机会太少 → 适当降低信号强度

4. **做空要更谨慎**
   - 建议做空信号强度 ≥ 2.5%
   - 或暂时禁用做空，只做多

### 触发条件 (已优化)

V3策略同时满足以下**所有条件**才会挂单：

1. **15M连续3根K线EMA9同向移动** ⚡ 优化

   - 做空：连续3次下降，每次变化 ≥ **0.5%**

2. **15M当前信号强度 ≥ 2.0%** ⚡ 优化
   - 计算：`|EMA9 - EMA26| / EMA26 × 100%`
   - 确保趋势足够强劲 (从1.0%提高到2.0%)

3. **15M EMA方向与持续趋势一致**
   - 做多：EMA9 > EMA26

4. **1H方向确认**

   - 使用统一的开仓冷却检查

### 限价单设置 ⚡ 动态优化

V3策略支持**动态限价单**（推荐）或固定限价单：

**动态限价单** (2026-01-15 新增):
- 根据**信号强度**、**市场波动率(ATR)**、**趋势阶段**自动计算
- 偏移范围：0.5% - 2.5%

**动态计算公式**:
```
基础偏移 = 根据信号强度:
  - 强信号(≥2.5%): 0.8%
  - 中信号(2.0-2.5%): 1.0%
  - 弱信号(1.5-2.0%): 1.3%

ATR调整 = min(ATR% / 1.0%, 1.5倍)

趋势阶段调整:
  - 早期趋势: × 1.2
  - 中期趋势: × 1.0
  - 成熟趋势: × 0.8

最终偏移 = 基础偏移 × ATR调整 × 趋势阶段调整
```

**示例**:
- BTC 信号强度2.8%, ATR 0.8%, 成熟趋势 → 限价单0.8% × 0.8 × 0.8 ≈ 0.5%
- ETH 信号强度2.0%, ATR 1.2%, 中期趋势 → 限价单1.0% × 1.2 × 1.0 = 1.2%
- SOL 信号强度1.8%, ATR 1.5%, 早期趋势 → 限价单1.3% × 1.5 × 1.2 ≈ 2.3%

**固定限价单** (向下兼容):
- **做多**: `market_minus_1` 或 `market_minus_2`
- **做空**: `market_plus_1` 或 `market_plus_2`

### ⚡ V3持仓期间趋势质量动态监控 (2026-01-15 新增)

> **核心问题**: "我们是以趋势入场，但不能确保这个趋势在持仓的过程中是否已经发生了重大变化，而我们没有及时的调整"

#### 监控逻辑

V3策略新增**趋势质量评分系统** (0-100分)，持续监控持仓期间趋势是否仍然有效，在趋势减弱或反转时及时平仓。
