# 服务说明

## 两个不同的调度器

### 1. `app/scheduler.py` - 数据采集调度器
- **功能**：采集市场数据（价格、K线、新闻等）
- **日志文件**：`logs/scheduler_*.log`
- **不执行策略**：只负责数据采集

### 2. `app/strategy_scheduler.py` - 策略执行调度器 ⭐
- **功能**：执行交易策略，检测信号，自动交易
- **日志文件**：`logs/scheduler_*.log`（可能和数据采集器共用）
- **执行策略**：会调用 `StrategyExecutor` 执行策略并记录命中信息

## 如何确认运行的是哪个？

### 检查日志中的启动信息

**数据采集调度器启动时会显示：**
```
初始化 binance 采集器
开始采集币安合约数据...
```

**策略执行调度器启动时会显示：**
```
初始化策略执行器...
  ✓ 策略执行器初始化成功
🔄 策略实时监控服务已启动（间隔: 5秒，实时响应新K线）
```

## 需要运行哪个？

**要记录策略命中信息，需要运行：**
```bash
python app/strategy_scheduler.py
```

**如果只运行了数据采集器，策略不会执行，也不会记录命中信息！**
