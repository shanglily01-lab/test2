## 4. 核心模块设计

### 4.1 信号评分引擎 (SignalScoreV2Service)

```
输入: symbol, timeframe
  ↓
读 coin_kline_scores (预计算评分,每5分钟更新)
  ↓
加载 signal_scoring_weights (strategy_type)
  ↓
14 个信号组件加权求和
  ↓
返回 long_score (0~217), short_score (0~243)
  ↓
过滤: score < threshold 或 score > max_threshold(150) 拒绝
```

**14 个信号组件 (strategy_type='default')**：

| 组件 | 多头权重 | 空头权重 | 方向 |
|------|---------|---------|------|
| RSI_OVERSOLD | 高 | - | 超卖反弹 |
| RSI_OVERBOUGHT | - | 高 | 超买回落 |
| MACD_BULLISH | 高 | - | MACD 金叉 |
| MACD_BEARISH | - | 高 | MACD 死叉 |
| BB_LOWER_TOUCH | 中 | - | 布林带下轨 |
| BB_UPPER_TOUCH | - | 中 | 布林带上轨 |
| EMA_BULLISH | 中 | - | EMA 多头排列 |
| EMA_BEARISH | - | 中 | EMA 空头排列 |
| VOLUME_SURGE | 中 | 中 | 成交量放大 |
| KDJ_OVERSOLD | 中 | - | KDJ 超卖 |
| KDJ_OVERBOUGHT | - | 中 | KDJ 超买 |
| PRICE_ACTION_BULL | 中 | - | 价格动作看多 |
| PRICE_ACTION_BEAR | - | 中 | 价格动作看空 |
| TREND_STRENGTH | 高 | 高 | 趋势强度 |

### 4.2 Big4 趋势检测 (Big4TrendDetector V3 算法)

```python
# 核心算法 (_analyze_symbol_v2)
取最近 16 根 15M K线 (=4 小时窗口)
bearish_count = 阴线数
bullish_count = 阳线数
price_change_pct = (close[-1] - open[0]) / open[0] * 100

# BTC 特别阈值 0.2%,其他币 0.3%
bullish_threshold = 0.2 if symbol == 'BTC/USDT' else 0.3
bearish_threshold = -0.2 if symbol == 'BTC/USDT' else -0.3

if bearish_count >= 11 and price_change_pct <= -0.5:
    signal = STRONG_BEARISH
elif bearish_count >= 9 and price_change_pct <= bearish_threshold:
    signal = BEARISH
elif bullish_count >= 11 and price_change_pct >= 0.5:
    signal = STRONG_BULLISH
elif bullish_count >= 9 and price_change_pct >= bullish_threshold:
    signal = BULLISH
else:
    signal = NEUTRAL

# 加权合并 (BTC 50% + ETH 30% + BNB 10% + SOL 10%)
# 写入 big4_trend_history 表
```

**reason 字段格式**: `15M强空头(11/16阴线,区间-1.59%)`

### 4.3 紧急干预 (_detect_emergency_reversal)

```
【1H 级别检测】
  最近 4 小时 BTC/ETH/BNB/SOL 任一币:
    跌幅 > 5% → BOTTOM_BOUNCE (触底反弹)
    涨幅 > 5% → TOP_REVERSAL (顶部反转)

【15M 级别检测】(深 V 触底)
  1H 长下影线 > 3% + 后续 15M 连续 3 根阳线
    → BOTTOM_BOUNCE

【触发后写入】
  emergency_intervention 表:
    intervention_type, expires_at, block_long, block_short
  bounce_window 表:
    window_end = NOW() + 45min,允许全市场做多

【动态解除】
  反弹窗口结束后,Big4 已反弹 2%+ → 解除做空封禁
  反弹窗口结束后,Big4 已回调 2%+ → 解除做多封禁
```

### 4.4 震荡市检测 (_detect_choppy_market)

```
过去 4 小时检测 Big4 信号变化次数
  ↓
反复切换 BULLISH ↔ BEARISH ↔ NEUTRAL → is_choppy=True
  ↓
主循环检测: is_choppy + 非 STRONG 信号 → 跳过本轮开仓
例外: STRONG_BULLISH / STRONG_BEARISH 仍允许顺势开仓
```

### 4.6 SmartExitOptimizer (平仓监控) [v3.4 校正 · 2026-06-21 补充]

**适用范围**: smart_trader 主策略、S1-S9 等 **非** AI 模拟 source。**排除**四路 `*_midline_*`（中线由 `position_sl_tp_monitor` 独立监管）。

**模拟 AI 仓**（探索/预测/战术）: SmartExit **不**作为主 SL/TP 通道；`crypto-app-main` 内 `position_sl_tp_monitor`（1s）负责硬 SL/TP + ai-trail-tp + planned_close。

**实际优先级顺序** (来自 _check_exit_conditions + _smart_exit):

```
任何时候:
  P1. 止损价触发 (stop_loss_price)
  P2. 止盈价触发 (take_profit_price)
  P3. ROI ≤ -15% 极端亏损兜底 [v3.3 放宽: 原 -10%, 5x 杠杆下价格仅 2% 即触发]
       开仓 30 分钟内豁免 (避开建仓初期插针)

多策略 S1-S9 持仓: 仅 P1/P2/P3,后续步骤跳过

主策略 (持仓 ≥60 分钟后):
  P4. 趋势反转:5M K 线 LONG ≥11/16, SHORT ≥9/16 根逆向 + 亏损 → 平仓
  P5. [v3.3 启用] 移动止盈: 峰值价格收益 ≥ 6% 激活,从峰值回撤 ≥ 1% 平仓
       移动止损暂未启用

planned_close_time 前 30 分钟启动智能平仓窗口:
  P6. T-30: 初始化 PriceSampler 采样
  P7. T-20: 完成 10 分钟价格基线
  P8. T-20 ~ T+0: 等待最优价格平仓
  P9. T+0: 强制平仓 (planned_close_time 必须执行)

兜底:
  P10. ROI ≥ +25% 超高盈利全部平仓
  P11. ROI ≤ -15% 巨额亏损全部平仓 [v3.3 与 P3 同步]
```

**多策略 S1-S9 跳过 P4** (依赖固定 SL/TP + planned_close_time)。

**v3.4 保护层关系 (5x 杠杆下)** — SL/TP 按信号评分动态调整 (C5):

| 阈值 | 价格变化 | ROI | 备注 |
|------|----------|------|------|
| 极端止损 P3 | -3% | -15% | 开仓 30 min 豁免 |
| 固定 SL | -2% | -10% | system_settings.stop_loss_pct |
| 入场价 | 0% | 0% | - |
| 移动止盈激活 | +6% | +30% | 记录峰值 |
| 移动止盈触发 | 峰值回撤 1% | 回撤 5% | SmartExitOptimizer |
| 动态 TP | 6%~10% | 30%~50% | 按信号评分 C5:高分缩TP,低分放宽 |
| 固定 TP 兜底 | +8% | +40% | system_settings.take_profit_pct |
| 超高盈利全平 P10 | +5% (5x) | +25% | 兜底 |

**Gemini / DeepSeek 模拟仓持仓顾问** [2026-06-21 更新]:
- 文件: `gemini_position_advisor.py` / `deepseek_position_advisor.py`
- 调度: `crypto-scheduler` **每 5min** + smart_trader 主循环 5min 兜底
- 复审: 浮盈 ROI>0 → **5min/仓**；其余 **15min/仓**；浮盈转亏 → urgent 立即再审
- 路由: `gemini_explore/predict` → Gemini；其他 → DeepSeek；**中线四策略 excluded**
- 15m K 线主审；盈利 sell 门槛 ROI **+5%**；程序化 `_temper_*` 复核
- sell → 关模拟仓；`live_close_enabled=1` 时平映射实盘
- kill switch: `gemini/deepseek_position_advisor_enabled`

**position_sl_tp_monitor** [2026-06-21]:
- 进程: `crypto-app-main`（async 1s 轮询）
- AI 硬 SL/TP + **ai-trail-tp**（peak 价格 +3% 后回撤 1%）

### 4.10 市场预测器 (MarketPredictor) [v3.1 校正]

```
每 6 小时由 app/main.py 调度:
  run_all(symbols):
    ① 平掉到期的真实模拟单 (planned_close_time 触发)
    ② 结算上一轮虚拟单 (prediction_backtest, 持仓 ≥ 5.5H)
    ③ 对每个 symbol 调用 analyze() 生成预测
       - 72H 1H K线 + 96根 15M K线
       - EMA9/26, RSI, ADX, MACD, 量能扩张
       - confidence 0-100
    ④ 开新虚拟单 (confidence ≥ 40, 100U × 5x)
    ⑤ 开真实模拟单 (confidence ≥ 70, source='PREDICTOR')
       ★ 代码无数量上限 (v3.0 文档错写"最多 5 单")
       默认 400U,黑名单 Level1/2 → 100U/50U
    ⑥ 实盘同步 (需 live_trading_enabled=1 + 在 Top50)
```

**S8 (顶部反转做空) 关键设计**:
- 入场最严: 12 天上市 + 48h pump >= 80% + 6 根 1h 无新高 + 信号 ≤6H + 现价 > pump 起点低 + 从峰回落 < 50% + 24H 跌幅 > -15% + 3H 15m 区间位置 ≥ 10%

- 不在 7d 低点接刀 (区间位置门槛)
- kill switch `s8_topshort_enabled` 默认 OFF

**S9 (Gemini AI 抄底反转) 关键设计**:

- Gemini 返回 short 自动降级 skip (LONG-only)

### 4.12 实盘对账 (reconcile_live_positions) [v3.1 新增]

每 5 分钟触发,双向对账:

```python
# 方向 1: DB OPEN 但交易所已平 → 自动 CLOSED
db_opens = SELECT FROM live_futures_positions WHERE status='OPEN'
exchange_positions = engine.get_open_positions()

for row in db_opens:
    if (symbol, side) not in exchange_set:
        # 价格兜底链 (2026-05-14 修复):
        close_p = get_current_price(symbol)              # WS 优先
              or exchange_price_map.get(symbol)           # 当前持仓 mark_price
              or kline_data 最新 5m close (陈旧带告警)   # 第三层兜底

        # 计算 PnL
        if close_p and entry_price and quantity:
            pnl = (close - entry) * qty  (LONG)
                or (entry - close) * qty  (SHORT)
        else:
            logger.error("PnL 计算失败,这是错误数据")

        UPDATE live_futures_positions SET
            status='CLOSED',
            close_reason='reconcile_closed',
            close_price=close_p,
            realized_pnl=pnl

        # 关联模拟单也标 closed
        if paper_position_id:
            UPDATE futures_positions SET status='closed'

# 方向 2: 交易所有但 DB 没有 → 补录
missing = exchange_set - db_open_set
if missing:
    engine.sync_positions_from_binance(account_id)
    # 写入 live_futures_positions, source='binance_sync'
```

**binance_sync 的意义**: 用户在 Binance app/网页**直接开仓**后,5 分钟内被系统同步进 DB。**这绕过了 Top50/Big4/评级所有过滤**,因为不是系统主动开仓。

### 4.13 反弹窗口 (bounce_window) [v3.1 新增]

```
紧急干预触发 BOTTOM_BOUNCE 后:
  bounce_window 表写入 window_end = NOW() + 45min
  ↓
smart_trader_service 主循环检测到 bounce_window 活跃:
  → 跳过常规信号过滤
  → 对全市场所有 Top50 交易对开多
  → window_end 后自动结束
```

### 7.1 自动开仓数据流 (主策略)

```

     ↓
SmartDecisionBrain.scan_all(big4_result)

  → 每 symbol 调用 analyze(symbol, big4_result)
    → 读 coin_kline_scores
```

### 7.3 实盘对账数据流

```

reconcile_live_positions():
  for each active api_key:
    exchange_positions = engine.get_open_positions()

    # 方向 1: DB OPEN → 交易所已平
    for db_row in OPEN positions:
      if not in exchange_set:
        close_p = WS → exchange_map → 5m K线 (3 层兜底)

      sync_positions_from_binance() → source='binance_sync'
```

### 11.4 PID 文件锁 (v3.2 新增)

```python
# app/utils/pid_lock.py
def acquire_pid_lock(service_name, pid_dir='logs'):
    pid_file = f'logs/{service_name}.pid'
    if pid_file.exists():
        old_pid = read(pid_file)
        if _process_alive(old_pid):
            print(f"{service_name} 已在运行 PID={old_pid},退出")
            sys.exit(1)
        else:
            pid_file.unlink()  # 旧 PID 进程已死,清理
    pid_file.write(os.getpid())
    atexit.register(cleanup)  # 进程退出自动删 PID 文件
```

**集成位置**: 5 个入口脚本顶部 `if __name__ == '__main__':` 之后:
