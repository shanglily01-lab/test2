## 1. 测试策略

### 1.1 测试范围

| 测试类型 | 覆盖模块 | 优先级 |
|---------|---------|-------|
| 功能测试 | 信号评分、Big4检测、开平仓逻辑、多策略 S1-S7、实盘对账、ETF/金库同步 | P0 |
| API 接口测试 | 所有 REST API (21 模块,60+ 端点) | P0 |
| 集成测试 | 模拟盘完整交易流程、每日复盘、reconcile_live_positions | P1 |
| 性能测试 | 信号扫描延迟、API 响应、WS 延迟 | P1 |
| 安全测试 | 认证鉴权、SQL 注入防护、密钥脱敏 | P1 |
| 回归测试 | 每次代码改动后核心路径验证 | P0 |
| Bug 修复回归 | 已修复 bug 的回归用例 (1m K线 fallback、reconcile_closed PnL) | P0 |

### 1.2 测试原则

- 所有涉及实盘的测试必须在模拟盘模式下执行 (`live_trading_enabled=0`)
- 数据库操作使用测试数据库,不污染生产数据
- 修改代码后必须先运行语法检查:`python -c "import ast; ast.parse(open('file.py',encoding='utf-8').read())"`
- 每次测试前确认服务状态 (避免同时运行多个实例)
- **代码改动后必须重启对应进程** (Python 不热重载)

### 1.3 测试环境

| 环境 | 说明 |
|------|------|
| 开发测试 | 本地 Windows 10,连测试 DB,live_trading_enabled=0 |
| 模拟盘 | 远程服务器,真实数据,live_trading_enabled=0 |
| 生产 | 远程服务器,live_trading_enabled=1,谨慎操作 |

## 8. 已知缺陷与注意事项

### 8.1 已知限制 [v3.1 更新]

| 问题 | 说明 | 规避方法 |
|------|------|---------|

### 8.2 测试陷阱 [v3.1 更新]

| 陷阱 | 说明 |
|------|------|
| 实盘持仓查错表 | 实盘在 live_futures_positions,模拟在 futures_positions |
| system_settings 列名 | 是 setting_key/setting_value,不是 key/value |
| trading_symbol_rating 无 exchange 列 | 查询时不要加 exchange 条件 |
| symbol 格式 | DB 是 'IO/USDT' 带斜杠,不是 'IOUSDT' |

| 多进程独立 | 修改 app/main.py 不影响 smart_trader_service.py |
| source 命名 | smart_trader 才是主策略实际写入值 (不是 signal_confirm) |
| SL/TP 实际值 | 从 system_settings 读取 (当前 2%/3%),非 config.yaml |
| max_hold_hours 实际值 | 当前 3 小时,非文档暗示的 4/8 |
| binance_sync 旁路 | 用户手动 Binance app 操作不受 Top50/Big4 过滤 |
| PREDICTOR 无单数上限 | 代码注释明确"不限数量",非文档说的 5 单 |
