**版本**: v3.4
**日期**: 2026-05-21
**状态**: 正式版 (基于实际代码校正)
**修订**: v3.3 → v3.4 信号评分与风控系统性优化 (11项改进),见末尾"v3.4 修订记录"

## 1. 项目背景与目标

### 1.1 背景

加密货币市场 7×24 小时不间断运行,价格波动剧烈,人工监盘效率低且容易受情绪影响。本系统旨在构建一套全自动量化交易平台,基于技术信号、市场趋势和智能风控,在 Binance 交易所自动执行合约交易。

### 1.2 目标

- 实现 U 本位合约、币本位合约的全自动信号扫描与开平仓
- 提供 Web 管理界面,支持实时监控、参数调整与回顾分析
- 通过信号评分、Big4 趋势过滤、Top50 筛选等多道关卡控制风险
- 支持模拟盘与实盘切换,降低实盘测试风险
- 通过每日自动复盘与权重优化,持续改进策略表现

### 1.3 范围

| 模块 | 包含 | 不包含 |
|------|------|--------|
| 交易所 | Binance 合约 | 其他交易所、现货 |
| 合约类型 | U本位合约、币本位合约 | 期权 |
| 自动化 | 信号检测、开平仓、风控、复盘 | 人工干预执行 |
| 数据来源 | Binance K线、Farside ETF、BitcoinTreasuries | 链上数据实时监控 |

## 3. 功能需求

### 3.1 自动交易引擎

#### 3.1.1 U 本位合约主策略 (smart_trader_service.py)

**FR-U01** 信号扫描
- 每 5 分钟扫描白名单交易对 (SmartDecisionBrain.scan_all)
- 对每个交易对计算多空评分 (LONG 满分 217 / SHORT 满分 243)
- 根据 Big4 市场趋势决定开仓阈值
- **实际写入 `futures_positions.source = 'smart_trader'`** (代码预留 signal_confirm / trend_follow 但未启用)

**FR-U02** 开仓阈值 (Big4 过滤器开启时, v3.4 放宽)
- STRONG_BULLISH：LONG=60+从众惩罚, SHORT=60
- STRONG_BEARISH：LONG=100 (封死), SHORT=60
- BULLISH (signal_strength ≥ 50)：LONG=60+从众惩罚+5 (即65), SHORT=60
- BEARISH (signal_strength ≥ 50)：LONG=100 (封死), SHORT=60
- NEUTRAL：LONG=65+15 (即80,且要求 score≥80), SHORT=60
- max_threshold=150：分数超过 150 拒绝 (防追涨杀跌)
- 实盘模式：交易对必须在 top_performing_symbols Top50 内

**FR-U03** 风控关卡 (16 道 + 3项优化, v3.4 扩展)
- 持仓数量限制 (MAX_POSITIONS=50,可通过 system_settings.max_positions 调整)
- 同币种同方向防重复
- 信号黑名单 (signal_blacklist)
- 评级黑名单 (rating_level >= 3 禁止)
- **资金费率动态阈值 (v3.4: 24h均值±2σ替代固定±0.1%)** → [FUNDING_VETO]
- 反向信号过多 (≥3 个拒绝)
- Big4 方向封锁
- **信号拥挤度惩罚 (v3.4: 同向≥4个组件时扣溢出分)** → [CROWDING]
- 从众探测器 (≥80% 同向时阈值 +10) → [HERDING]
- **V2数据过期保守模式 (v3.4: 数据>15min时阈值+10)**
- 连续亏损习惯化 (最多 5 次后阈值×0.6→相当于提升至 100 分) → [HABITUATION]
- 不应期冷却 (同币同方向 120 分钟内不重复) → [REFRACTORY]
- 信号衰减检测 (亏损时若评分反转则平仓) → [SIGNAL_DECAY]
- 盈利熔断 (6 小时盈利 > 1000U 自动暂停) → [PROFIT_GUARD]
- 亏损熔断 (3 小时亏损 > 300U 自动暂停) → [LOSS_GUARD]
- 震荡市检测 (is_choppy=True 非强信号时暂停) → [CHOPPY_MARKET]

**FR-U04** 建仓执行 (v3.4 增强)
- 使用 SmartEntryExecutor (价格采样 V1,15 分钟采样,一次性建仓)
- 杠杆：LEVERAGE=5
- 保证金：Level 0 = 400U,Level 1 = 100U,Level 2 = 50U,Level 3 = 不开仓
- **差异化仓位大小 (v3.4): 按评分动态调整,弱信号x0.6,强信号x1.2**

**FR-U05** 平仓管理 (SmartExitOptimizer) [v3.4 多重增强]
- 止损/止盈：**实际从 system_settings 读取,当前值 stop_loss_pct=0.02 / take_profit_pct=0.03**
  - **v3.4 新增动态SL/TP**: 按信号评分自动调整,弱信号SL放宽至5%/TP缩至6%,强信号SL缩至2%/TP放至10%
  - 代码默认 SL 2% / TP 5% (BTC_MOMENTUM 与 PREDICTOR 内部默认)
  - 主策略 SL 2% / TP 6% (config.yaml 写)
  - **运行时以 system_settings 为准**,管理员可随时改
- **v3.4 退出原因枚举**: 每种平仓reason映射到短码(SL/TP/TREND_REV/TRAILING_TP等),写入 notes 和 close_reason
- **v3.4 监控频率递减**: <30min:1s/次, 30min-2h:3s/次, 2h-4h:5s/次, >4h:10s/次
- 最大持仓时间：从 system_settings.max_hold_hours 读取,**当前实测值 3 小时**
- ROI ≤ -10% 极端亏损兜底立即平仓
- ROI ≥ 25% 超高盈利立即全部平仓
- 趋势反转检测：60 分钟最小持仓后,LONG ≥11/16 / SHORT ≥9/16 根 5M K 线逆向 → 平仓
- 多策略 S1-S7 持仓**跳过**动态趋势反转、K 线衰减检测,只用固定 SL/TP + planned_close_time
- 智能平仓窗口：planned_close_time 前 30 分钟启动 PriceSampler 采样最优价
- 移动止盈：**代码注释为"暂时关闭"** (0.5% 阈值被插针频繁触发)

#### 3.1.2 币本位合约交易 (coin_futures_trader_service.py)

**FR-C01** 信号扫描
- 每 5 分钟扫描币本位合约交易对
- 使用 CoinFuturesDecisionBrain 决策逻辑

**FR-C02** 开仓条件
- 基础阈值：threshold=60
- Big4 偏空市场时 LONG 阈值动态上调至 68
- Big4 = NEUTRAL：只过滤 LONG,SHORT 仍可执行 (2026-04-05 修复)

**FR-C03** 特色功能
- detect_big4_bottom_reversal：Big4 触底反转检测,触发后允许 LONG 跨过封锁
- detect_big4_top_reversal：Big4 见顶反转检测,触发后允许 SHORT 跨过封锁
- check_breakout：BreakoutSystem 破位系统集成
- 紧急干预 (emergency_intervention 表) block_long / block_short 强制控制方向

**FR-C04** 建仓执行
- 杠杆：LEVERAGE=5,MAX_POSITIONS 不限 (999)
- 保证金：正常 400U,黑名单 100U
- 盈亏熔断阈值低于 U本位 (盈利 200U/6H,亏损 100U/3H)

#### 3.1.3 BTC 动量跟随策略 (app/services/btc_momentum_trader.py)

**FR-B03** 价格数据 (2026-05-14 修复)

**FR-B04** 持仓管理

#### 3.1.4 市场预测器 (app/services/market_predictor.py)

**FR-P01** 调度

**FR-P02** 虚拟单 (prediction_backtest 表)
- confidence ≥ 40 对所有交易对开虚拟单
- 100U × 5x 杠杆,5.5 小时后自动结算 PnL
- 仅记录用于回测,不实际下单

**FR-P03** 真实模拟单 (futures_positions, source='PREDICTOR')

**FR-P04** 实盘同步

**FR-M03** S1-S9 各策略详细入场算法

##### S1 早期做多 (`scan_s1_early_long`)
1. 持仓数 < S1_MAX_POSITIONS (=999)
2. Big4 不在 (BEARISH, STRONG_BEARISH)
3. 候选池: `_get_candidate_symbols(min_abs_change=3.0)` 即 24H 涨跌幅绝对值 ≥ 3%
4. 同 symbol 在 S1-S9 任意已开仓则跳过
5. 1H K 线 (30 根)
   - RSI 最新值 ∈ [25, 52]
   - RSI 上一根 < 最新值 (上升)
6. 4H K 线 (40 根),MACD histogram (12,26,9)
   - 最新 3 根 hist 严格递增 (h1 < h2 < h3)
   - h1 (倒数第 3 根) < 0.002 (确认仍在低位)
7. 日 K 线 (25 根)
   - MA20 计算
   - 日收 ∈ [MA20 × 80%, MA20 × 105%]
   - 日 volume ≥ 近 7 日均量 × 0.9
8. 全部满足 → 开 LONG;reason 含 RSI / MACD / 价/MA20 / 量比

##### S2 无量回调做多 (`scan_s2_pullback_long`)
1. 持仓数 < S2_MAX_POSITIONS (=999)
2. Big4 不在 (BEARISH, STRONG_BEARISH)
3. 候选池: `min_abs_change=8.0`
4. 1H K 线 (52 根)
   - 48 根内 (high-low)/low ≥ 12% (有足够振幅)
   - (48 根 high - 当前 close) / 48 根 high ∈ [15%, 38%] (从高点回落)
5. 15m K 线 (30 根)
   - RSI ∈ [30, 58]
   - RSI 最新 > 上一根 (企稳上升)
6. 全部满足 → 开 LONG

##### S3 顶部做空 (`scan_s3_top_short`)
1. 持仓数 < S3_MAX_POSITIONS (=999)
2. Big4 ≠ STRONG_BULLISH
3. 候选池: `min_abs_change=15.0` (大涨幅币种)
4. 1H K 线 (52 根)
   - RSI 最新 > 70
   - RSI 顶背离: 当前 RSI < 最近 6 根 RSI 最高 (说明 RSI 已开始下行但价格还在涨)
   - 现价从 48H high 回落 ∈ [3%, 15%]
   - 48H 涨幅 = (48H high - 48 根前收) / 48 根前收 > 20%
5. 4H K 线 (40 根) MACD hist 下行 (h3<h2 或 h2<h1)
6. 最近 3 根 1H K 线 ≥ 2 根阴线 (close < open)
7. 全部满足 → 开 SHORT

##### S4 反弹动能衰竭做空 (`scan_s4_rebound_short`)
1. 持仓数 < S4_MAX_POSITIONS (=999)
2. **无 Big4 过滤** (反弹是市场结构信号,与 Big4 无关)
3. 候选池: `min_abs_change=8.0`
4. 14 日 1H K 线 (14×24+2 = 338 根)
   - two_week_high = 14 日 high 最大值
   - 当前价 / two_week_high ∈ [50%, 85%]
   - 7 日 low 计算; (two_week_high - 7日low) / two_week_high ≥ 15% (有实质性回落)
   - 现价 > 7日low × 1.05 (从低点已反弹 ≥ 5%)
5. 最近 52 根 K 线计算指标,三个空头条件:
   - 条件 A — MACD hist 下行: h3<h2 或 h2<h1
   - 条件 B — RSI < 65 且 RSI 下行
   - 条件 C — 量萎缩: 近 10 根中,上涨 K 均量 < 下跌 K 均量
   - **A/B/C 三选二**
6. 满足 → 开 SHORT

##### S5 大币 4H 超卖反弹做多 (`scan_s5_large_oversold`)
1. 持仓数 < 3
2. Big4 ≠ STRONG_BEARISH
3. **白名单**: [BTC, ETH, SOL, BNB, XRP]
4. 4H K 线 (40 根)
   - RSI < 32 (超卖)
   - RSI 当前 > 上一根 (回升,不再下落)
5. 日 K 线 (25 根)
   - 当前日收 < MA20 (均值回归空间)
6. 满足 → 开 LONG

##### S6 量能异动做多 (`scan_s6_vol_spike`)
1. 持仓数 < 5
2. Big4 不在 (BEARISH, STRONG_BEARISH)
3. **排除大市值 17 个 + FIO/USDT** (`S6_EXCLUDE_SYMBOLS`)
   - 大币量能信号失真;FIO 回测 10 天 8% 胜率
4. 1H K 线 (50 根)
   - 48 根均量 (vol_base)
   - 当前量 / vol_base ∈ [1.2, 5.0]
   - 12 根 high vol / vol_base > 3.5 (近 12H 出现量能爆发)
   - RSI ∈ [28, 55]
   - 现价 / 1H_MA20 ∈ [75%, 108%]
   - 过去 3 根 1H 涨幅 < 5% (未追高)
5. 满足 → 开 LONG

##### S7 小币均线支撑反弹做多 (`scan_s7_ma_support`)
1. 持仓数 < 5
2. Big4 不在 (BEARISH, STRONG_BEARISH)
3. 排除大市值 (同 S6 列表)
4. 1H K 线 (30 根)
   - 20H 简单均线 (最后 21 根 close mean)
   - 现价 / 20H_MA ∈ [82%, 95%] (在均线下方但未深跌)
   - 当前 K 线: close > open 且 close > 前根 close (阳线 + 高于前收)
   - 当前量 ≥ 10 根均量 × 1.2 (量能确认)
5. 满足 → 开 LONG

##### S8 顶部反转做空 [v3.2] (`scan_s8_topshort`)
1. **kill switch**: `s8_topshort_enabled` 必须 ON (默认 OFF)
2. 持仓数 < 3
3. Big4 ≠ STRONG_BULLISH
4. 候选池: `min_abs_change=50.0` (大涨幅币种)
5. 上市 ≥ 12 天 (`_s8_has_min_listed_history`,15 分钟缓存)
6. 1H K 线 (LOOKBACK 48 + NO_NEW_H 6 + 20 = 74 根)
   - 倒序找最近一个满足 pump 的 i:
     - pump_i = (highs[i] - min(lows[i-48:i])) / min ≥ 80%
     - 之后 6 根 1H 全部 high < highs[i] (无新高)
     - i + 6 的 K 线 open_time 距今 < 6H (信号未老化)
   - 现价 > pump 起点低点 (信号未失效)
   - (峰值 - 现价) / 峰值 < 50% (跌得不太狠,不接刀)
7. `price_stats_24h.change_24h` > -15% (24H 未崩盘)
8. `_s8_entry_position_check`: 3H 内 15m K 线 (12 根) 计算区间位置 (cur - lo) / (hi - lo) × 100 ≥ 10% (现价不在低点)

### 3.4 Big4 趋势检测 (app/services/big4_trend_detector.py V3 算法)

**FR-G01** 单币计算 (_analyze_symbol_v2)
- 取最近 16 根 15M K 线 (=4 小时窗口)
- 计算 bearish_count (阴线数), bullish_count (阳线数), price_change_pct (区间涨跌)
- BTC 特别阈值:0.2%,其他币:0.3% (BULLISH/BEARISH 判定时)

**FR-G02** 单币信号等级

| 条件 | 信号 |
|------|------|
| 阴线≥11 且区间≤-0.5% | STRONG_BEARISH |
| 阴线≥9 且区间≤-0.3% (BTC -0.2%) | BEARISH |
| 阳线≥11 且区间≥+0.5% | STRONG_BULLISH |
| 阳线≥9 且区间≥+0.3% (BTC +0.2%) | BULLISH |
| 其余 | NEUTRAL |

reason 字段格式:`15M强空头(11/16阴线,区间-1.59%)`

**FR-G03** Big4 合成
- 权重:BTC 50% + ETH 30% + BNB 10% + SOL 10%
- 加权阈值 > 60% 触发 BULLISH / BEARISH;阴阳同时强势时取强者
- 写入 big4_trend_history 表

**FR-G04** 紧急干预 (_detect_emergency_reversal)
- 1H 级别:最近 4 小时 BTC/ETH/BNB/SOL 任一跌/涨 > 5% → 触底/触顶
- 15M 级别:1H 长下影线 (>3%) + 后续 15M 连续 3 根阳线 → 深 V 触底
- 触发后写 emergency_intervention 表 (block_long / block_short / expires_at)
- 配合反弹窗口 bounce_window:触底后 45 分钟内允许全市场开多

**FR-G05** 震荡市检测 (_detect_choppy_market)
- 过去 4 小时 Big4 信号反复切换 → is_choppy=True
- 主循环检测到 is_choppy 且非 STRONG 信号时跳过本轮开仓
