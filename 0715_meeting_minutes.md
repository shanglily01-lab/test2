## 一、会议主题

围绕发牌算法干预效果、用户生命周期与活跃画像、IP/国家/时区分析、SNG 与机器人投放、运营 Agent 能力、以及干预牌局回放看板需求进行复盘，并明确下一阶段需要补充的数据看板和产品化细节。

## 二、核心结论

1. 发牌干预对多类用户的胜率、下注强度和后续活跃有明显正向影响。
   - 新用户、流失用户、削峰填谷等场景中，干预与不干预相比，胜率提升大致在 20% 上下。
   - 新用户池子整体金币规模较小，因此金币增量不高，但真人用户会因扶持获得一定金币增加。
   - 对照测试显示，干预后未来 7 天平均在线时长、在线天数有提升，用户体验更好。

2. 流失用户不能等到“已经流失”后再干预，需要提前预判。
   - 已经低频登录、低频游戏后，再通过发好牌干预，效果会变弱。
   - 应基于登录频率、打牌手数、在线时长等下降趋势识别“潜在流失用户”，提前做扶持或激活。

3. 需要从“牌局视角”扩展到“单个玩家生命周期视角”。
   - 关注玩家从新注册、体验包、削峰填谷扶持、低活跃/潜流失扶持，到成为稳定用户的完整路径。
   - 建议建立“单个玩家系统干预专题”，展示某个玩家在平台完整周期内被干预、被扶持、标签变化、金币变化和活跃变化的过程。
   - 如果玩家在平台留存 3 个月以上，可初步视为稳定/忠实用户，重点分析其成长路径。

4. 活跃用户画像需要从单标签统计升级为“标签组合 + 玩家表现”的交叉分析。
   - 单个标签价值有限，需要把标签回归到玩家身上，看标签组合与在线时长、胜率、牌局表现、国家、游戏偏好之间的关系。
   - 优先拉出高活跃用户、Top 用户清单，再统计这些用户共同具备的标签、占比和相似度。
   - 该分析对后续运营、人群识别和获客方向有较高价值。

5. IP 国家库已经初步建立，但需要结合本地时间校验。
   - 已通过免费 IP API 查询存量 IP 对应国家，后续可按国家观察活跃情况。
   - 需要把用户在线时间转换为对应国家/地区本地时间，观察是否存在半夜高活跃等异常，以判断 IP 是否不准或存在代理/VPN。
   - 需要补充在线人数最多的时间段、热力图或统计图，为活动投放和桌内播报时机提供依据。

6. 看牌/必胜版机器人已小流量上线，但效果未达预期，需要继续 debug 和调参。
   - 当前看牌机器人会记录公共牌与手牌组合出的最大牌型，并根据自身牌力决定弃牌、跟注、拉扯等策略。
   - 金币场看牌机器人胜率约 30%，低于 V4/V5 约 40% 的水平；平均 BB、总输赢也未达到预期。
   - 必胜版机器人当前约 3%-5% 小流量触发，整体机器人约 20% 左右，必胜版比例不宜过高，需先把策略调稳。

7. 干预牌局回放需要产品化进入工作台。
   - 不只看成功案例，也要看失败案例，例如干预后玩家为何没有跟注、在哪一轮放弃、公共牌和下注过程如何。
   - 回放应能展示每一轮手牌、公共牌、下注/加注情况和最终结果，最好在一个页面内完整呈现，减少翻页。

## 三、分议题记录

### 1. 发牌干预与金币流动

- 新用户池子较小，因此发好牌带来的金币增量约千万级，不会特别高。
- 流失用户在干预后也会有金币增加，胜率提升接近 20%。
- 削峰填谷相关场景中，各组用户干预后的胜率提升基本在 20% 上下，平均每手下注量也有较高提升。
- 当前表格中“局数”不等同于“用户数”，因为同一个用户可能参与多个牌局或多个牌阶，后续需要明确干预影响的去重用户数。

### 2. 详情页与用户清单

- 现有表格需要增加二级钻取能力。
- 在详情页中展示：
  - 被干预用户清单；
  - 用户本周金币增减；
  - 干预前后的金币变化；
  - 削减最多的玩家；
  - 补充最多的玩家；
  - 影响最高的 Top 用户，例如 Top 10。
- 详情入口可通过“详情”按钮进入二级菜单。

### 3. 单个玩家生命周期专题

- 从玩家视角观察平台完整干预路径：
  - 新玩家进入平台时，干预比例是多少；
  - 一个月后干预比例如何变化；
  - 玩家自身打牌水平是否提升；
  - 是否逐步从需要扶持变为能稳定参与；
  - 新用户体验包、削峰填谷、潜流失扶持等动作如何作用于同一玩家。
- 该专题应展示玩家完整周期内的图表，帮助理解玩家如何从新用户成长为稳定用户。

### 4. 作弊用户打压

- 当前作弊用户样本较少，对整体金币影响不大，坏用户金币减少约 700 万级别。
- 需要补充作弊用户详情：
  - 哪些用户被识别为作弊；
  - 当前持有金币数量；
  - 历史金币来源与赢取情况；
  - 被干预后打牌局数是否下降；
  - 输赢情况是否下降。
- 初步数据看，作弊用户被干预后，打牌局数和输赢表现均有下降。

### 5. 短期发牌策略方向

- 对高余额用户可考虑提高削峰/干预比例，例如金币余额大于 1 亿或大于 1000 万的用户。
- Top 20 高余额用户中，目前仍有部分用户活跃，可优先观察这些活跃高余额用户。
- 新用户和流失用户当前效果较明显，可适度提高进入干预池的比例。
- 用户偏好牌型与牌库牌型匹配仍是中长期方向，需要对牌库和用户分别分析打标后再做匹配，不是一两天能完成的事项。

### 6. 潜流失用户识别

- 不应等用户低频或不登录后才定义为流失用户。
- 需要通过数据趋势提前预判：
  - 登录频率下降；
  - 打牌手数下降；
  - 在线时长下降；
  - 活跃天数减少。
- 可用活跃指标建模识别潜在流失用户，再进行适度扶持或激活。

### 7. 活跃用户画像

- 近 30 天在线时长中位数约 18 分钟，50% 以上用户在线时间在 20 分钟左右。
- 95% 或 99% 分位以上用户在线时长非常高，存在极端活跃用户，最高可到 400 多小时。
- 注册 7-30 天的成长期用户，相比新手期或成熟用户，在线时长更高。
- 国家维度初步看，印度尼西亚、埃塞俄比亚、德国、乌克兰等国家表现较好。
- 牌风/用户偏好维度中，部分 LAG、LP、偏激进或擅长某些打法的用户活跃占比更高。
- 输赢分布上，亏很多和赢很多的用户都更活跃，中间不输不赢用户活跃度较低。
- 游戏类型上，偏好现金桌的用户明显更活跃，在线时长和活跃用户占比更高。

### 8. 在线时段与运营活动

- 需要统计在线用户最多的时间段，可用热力图、统计图或饼图展示。
- 统计时要按 IP 国家换算本地时间，观察当地时间是否合理。
- 如果某国家在本地半夜大量在线，可能说明 IP 不准、用户使用代理，或地理归属需要修正。
- 活动投放、奖励播报、桌内随机奖励等运营动作，建议在人最多、牌桌最热闹的时间段进行。
- 人多时干预和奖励的热力叠加效果更明显，更容易产生刺激感和传播感。

### 9. SNG 场

- SNG 场近期量很少，例如最近一次在 6 月 29 日出现，一天只有几十手到上百手。
- 当前 SNG 场没有调用机器人，因此机器人相关数据中暂未体现 SNG 内容。
- 后续如果 SNG 有量，需要单独纳入看板观察。

### 10. 看牌/必胜版机器人

- 当前 Pro/看牌版本已小流量上线。
- 实现逻辑：
  - 发牌后计算每个玩家手牌与公共牌可组成的最大牌型；
  - 将每个玩家当前牌局最强牌型写入线上 cache；
  - 机器人触发看牌机制后，判断自身牌力是否全场最高；
  - 牌力弱时倾向弃牌或做拉扯；
  - 牌力强时倾向跟注、诱导对手继续跟注。
- 当前问题：
  - 金币场看牌机器人胜率约 30%，低于 V4/V5 约 40%；
  - 平均 BB、总输赢未达预期；
  - 近两周/半周数据仍存在波动；
  - 可能存在策略或实现 bug，需要继续 debug。
- 投放建议：
  - 继续保持小流量；
  - 必胜版机器人占比先按约 5% 触发；
  - 总体机器人比例约 20% 左右；
  - 调稳后再考虑进一步放量；
  - 必胜版比例不能过高，避免对整体生态造成过强干预。

### 11. 用户标签与高活跃用户分析

- 需要做一个用户标签总览页，观察标签覆盖人数、分布和交集。
- 不只看单个标签，而要看多个标签的交集/并集与玩家表现之间的差异。
- 建议优先做：
  - 高活跃用户清单；
  - Top 100、Top 300、Top 500 用户标签统计；
  - 高活跃用户共同标签；
  - 某些牌风标签占比是否达到 70%-80%；
  - 标签组合与在线时长、胜率、牌局表现、国家、游戏类型偏好的关系。
- 目标是识别“对平台运营效果好的用户类型/标签组合”，并反向理解其他用户为什么没有形成这些标签。

### 12. 运营 Agent

- Agent 目前采取“用到什么就补什么”的方式训练和补数据。
- 当业务方询问 Agent 拿不到数据时，应在后台补齐对应数据能力，再让 Agent 能回答。
- 后续继续按实际问题迭代，不需要单独大规模准备所有数据。

### 13. 干预牌局回放

- 需要把发牌算法干预的牌局回放接入工作台。
- 回放重点不是动画，而是能清楚看到每一轮牌局状态：
  - 六人桌每个玩家两张手牌；
  - 翻牌、转牌、河牌各阶段公共牌；
  - 每轮下注、跟注、加注、弃牌情况；
  - 干预对象是否成功跟注；
  - 成功或失败原因；
  - 最终结果。
- 展示方式建议参考游戏端现有回看功能，但在工作台中以一个页面完整列出，不要多页面来回切换。
- 可在页面中按第一轮、第二轮、第三轮等阶段做分段或底部阶段条，但核心信息应一眼可见。

- 看牌/必胜版机器人当前胜率、BB、投放比例等数字需以后端报表为准复核。
- 干预牌局回放需要确认是否直接复用游戏端回看数据结构，还是单独为工作台加工。
